# docs/reference

This directory holds non-normative reference material.

Use it for:
- design references
- comparisons
- implementation-state notes
- research material that informs plans or specs

Do not use it for:
- long-term canonical truth (`docs/specs/`)
- active implementation plans (`docs/plan/active/`)
- current handoff entrypoints (`docs/project-handoff.md`, session handoff docs)
- release instructions that are still directly operational (`docs/macos-release.md`)
