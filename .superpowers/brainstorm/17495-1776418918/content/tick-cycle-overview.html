Agent Kernel / Agent Assembly / Tick Cycle 总览

把你刚才要的 4 块内容放在一个页面里：概念图、表格、Tick 交互流程、LLM 交互流程。你可以一次性对照看。

1. 概念关系图
Agent Kernel + Agent Assembly + Tick Cycle
Agent Kernel（统一能力底座）
LLM invocation
调模型
context assembler
组上下文
tool / skill execution
执行工具与技能
memory retrieval
记忆检索
policy / approval
权限与审批

↓ 同一个 Kernel，上面挂不同装配
Loop Agent Assembly
prompt / context / tool / memory / policy / wakeup
Assistant Agent Assembly
prompt / context / tool / memory / policy / session
↓ 两者都跑同一个 Tick Cycle
Tick Cycle（唯一执行骨架）
Trigger → Bootstrap → Sense → Update Context → Assemble Request → Call LLM → Assistant Turn → Guard → Act → Observe → Re-enter Loop → Evaluate → Update Memory → Continue / Sleep / Respond / Escalate
2. Tick Cycle 表格
统一 Tick Cycle（表格视图）
节点 作用 LLM 对应
Trigger / Wakeup触发一次 tick尚未进 LLM
Bootstrap加载 Assembly尚未进 LLM
Sense收集输入/状态为请求准备原料
Update Context筛选上下文context 细分
Assemble Request组装请求system / user / tools
Call LLM调模型LLM request
Assistant Turn接收模型输出assistant
Guard / Policy Check动作守卫assistant → tool 前
Act执行动作tool execution
Observe读取结果tool result
Re-enter Loop必要时回到 LLMtool result → next assistant
Evaluate评估是否推进目标loop 后判断
Update Memory写记忆/摘要/计划不一定当前轮可见
Continue / Sleep / Respond / Escalate决定出口tick 结束
3. Tick 交互流程（外层）
Tick Cycle 外层流程
Trigger / Wakeup
→ Bootstrap
→ Sense
→ Update Context
→ Assemble Request
→ LLM Turn Loop
→ Evaluate
→ Update Memory
→ Continue / Sleep / Respond / Escalate

含义：这是完整的一次 tick。Loop Agent 和 Assistant Agent 共用这个骨架，差异不在流程，而在装配输入和出口状态。

4. 与 LLM 的直接交互流程（内层）
LLM Turn Loop 内层链路
Assemble Request
→ Call LLM
→ Assistant Turn
→ Guard / Policy Check
→ Act
→ Observe
→ Re-enter Loop（如需要）

含义：一个 tick 内部可以包含多轮 assistant → tool → assistant。真正与 LLM 直接往返的核心链路就在这里。

5. Context 的内部细分
Update Context 的组成
Context =
  Background
+ Current Input
+ World State Summary
+ History
+ Memory Index
+ Memory Detail (on demand)
+ Plan State
+ Tool / Skill Availability
+ Policy Constraints

Background

长期目标、persona、稳定 prompt 约束

Current Input

用户消息、定时触发原因、外部事件

World State Summary

当前真实业务状态快照

History

历史对话或最近事件摘要

Memory Index / Detail

先索引，后按需读详情

Plan / Tools / Policy

计划状态、能力可用性、权限边界
