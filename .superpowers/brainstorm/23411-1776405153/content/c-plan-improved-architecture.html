改进后的 C 方案：C-lite（最小内核 + 双模式 + 领域外壳）

这版不是“大一统 Agent OS”，而是吸收参考文档后的收缩版：世界状态外部化、memory 两阶段检索、Loop/Assistant 双模式分离、Recruiting 先作为第一个领域外壳。

为什么要改原始 C

文档原则 1

世界状态应外部化，优先 SQL / DB，不靠长 messages[] 维持。

文档原则 2

Memory 要“索引 + 详情”两阶段，不应该每次全量塞进 context。

文档原则 3

Loop Agent 和 Assistant Agent 运行模式不同，不能都建模成聊天机器人。

和原始 C 的差异

保留 C 的地方

  • 先抽统一 Kernel，而不是只做 Loop 特化层
  • Loop / Assistant 从第一天就是两个顶层 agent
  • policy / memory / skill lifecycle 统一抽象

对原始 C 的收缩

  • 不先抽大一统世界状态平台
  • 不先把招聘表和 runtime 全部中性化
  • 不把 Loop 错建成 chat session agent
