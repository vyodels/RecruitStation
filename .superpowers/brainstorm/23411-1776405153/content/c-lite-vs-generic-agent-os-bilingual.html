C-lite vs 通用 Agent OS：结构和执行过程对比

这版保留英文术语，并在后面补中文后缀解释，方便你对照理解。

一、结构对比
C-lite：最小内核 + 领域外壳
Loop Agent（常驻循环 Agent） / Assistant Agent（按需对话 Agent）
↓ 使用统一内核能力
Policy / Approval（权限与审批）
Memory Retrieval（记忆检索）
Skill Lifecycle（技能生命周期）
Agent Kernel（最小公共底座）
↓ 通过领域接口访问业务世界
RecruitingWorldStateProvider（招聘世界状态提供器）
RecruitingActionAdapters（招聘动作适配器）
Candidate / Application / JD 表（业务真实数据）
Recruiting Runtime / Queue / Run（招聘执行运行时）
Recruiting Memory Schemas（招聘记忆结构）

含义：招聘表继续是招聘表；Kernel 不拥有业务世界，只是访问和驱动它。

通用 Agent OS：大一统平台
Loop Agent / Assistant Agent / 其他 Agent（多个顶层智能体）
↓ 全都挂统一平台
Universal Agent Kernel（通用 Agent 内核）
Universal World State（通用世界状态层）
Generic Memory Store（通用记忆存储）
Unified Runtime（统一运行时）
Generic Entity Records（通用实体记录）
Generic Workflow State（通用工作流状态）
Generic Action Bus（通用动作总线）
Recruiting / Sales / Support 插件（领域插件）

含义：为了追求平台统一，容易诱导你把招聘表语义也抽掉，变成通用实体系统。

二、一次 Loop Tick 的时序对比
C-lite 执行时序
Scheduler（调度器）
  → wake Loop Agent（唤醒 Loop Agent）
Loop Agent（常驻 Agent）
  → Kernel: assemble stable context（组装稳定上下文）
  → RecruitingWorldStateProvider: query DB snapshot（查询数据库快照）
  → MemoryIndexProvider: list relevant memory indices（列出相关记忆索引）
  → LLM: decide next actions（决定下一步动作）
  → if need detail（如需详情）:
       MemoryDetailReader: read selected detail（读取详情）
       LLM: refine action（细化动作）
  → RecruitingActionAdapter / WorkerRuntime（招聘动作适配器 / 执行运行时）
       - enqueue run / call skill / write business data（入队执行 / 调 skill / 改业务数据）
  → EventBus / Approval / SleepPolicy（事件总线 / 审批 / 休眠策略）
  → return next wake condition（返回下次唤醒条件）

特点：世界状态直接来自招聘 DB；memory 是两阶段；Loop tick 是独立的，不依赖长消息历史。

通用 Agent OS 执行时序
Scheduler（调度器）
  → wake Loop Agent（唤醒 Loop Agent）
Loop Agent（顶层 Agent）
  → Universal Kernel: fetch generic world state（取通用世界状态）
  → WorldState Normalizer: map recruiting data into universal entities（把招聘数据标准化成通用实体）
  → Generic Memory Store: retrieve context（从通用记忆层取上下文）
  → LLM: decide next actions（决定下一步动作）
  → Generic Action Bus（通用动作总线）
       → Recruiting plugin rehydrates business meaning（招聘插件还原业务语义）
       → enqueue run / call skill / write data（入队执行 / 调 skill / 改数据）
  → Universal Event / Approval / Sleep（通用事件 / 审批 / 休眠）
  → return next wake condition（返回下次唤醒条件）

特点：先抽象成通用状态，再还原回招聘语义；理论更统一，但中间多了一层翻译层。

三、关键执行差异

C-lite

读世界：直接查招聘表

LLM 看见的是：招聘语义的真实快照

改数据：通过 recruiting action adapter（招聘动作适配器）

复杂度来源：领域逻辑本身

通用 Agent OS

读世界：先标准化成通用实体

LLM 看见的是：一层抽象后的通用状态

改数据：通过 generic action bus（通用动作总线）再映射回领域

复杂度来源：领域逻辑 + 抽象/翻译层

A

倾向 C-lite

统一 Kernel，但世界状态仍走招聘领域外壳

B

倾向通用 Agent OS

接受通用 world state / action bus / plugin 层
